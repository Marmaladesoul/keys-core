# Contributing

Thanks for your interest in `keys-core`. Please read this before opening a
pull request — the contribution model here is deliberately narrow.

## Code contributions are not accepted

**We do not accept external code contributions** (pull requests that add
or change source code, build configuration, or documentation).

This is not a comment on the quality of anyone’s work — it is a deliberate
licensing decision. Copyright in `keys-core` is held by a single author so
that the project can be licensed (and, where appropriate, dual-licensed or
relicensed) without having to track down and obtain agreement from every
past contributor. Accepting outside patches would fragment that copyright
and remove that freedom permanently.

Practically, that means:

- Pull requests that change code will be **closed unmerged**, regardless
  of merit. Please don’t invest effort in a PR expecting it to land.
- If you have an idea or have found a (non-security) bug, you are very
  welcome to **open an issue** describing it. Issues cost you nothing and
  help us improve the project — we just implement the change ourselves.

## Security reports *are* welcome

Security is the exception, and an important one. **We actively want to
hear about security vulnerabilities** in `keys-core`.

Please **do not** report security issues through public issues or pull
requests. Follow the process in **[SECURITY.md](SECURITY.md)** instead —
it covers private reporting, what to include, our response targets, and
coordinated disclosure.

## Bug reports and ideas (non-security)

For ordinary bugs and feature ideas, open a GitHub issue with:

- what you expected to happen and what actually happened;
- a minimal reproduction if you have one (a `keyhole` scenario is ideal);
- the component and version / commit involved.

We read every issue even though we may not act on all of them.

## Questions

General questions are best raised as a GitHub issue so the answer is
searchable for the next person. For anything sensitive, use the security
contact in [SECURITY.md](SECURITY.md).
